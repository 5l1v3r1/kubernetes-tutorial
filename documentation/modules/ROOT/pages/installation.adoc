[#download-tutorial-sources]
== Download Tutorial Sources
Before we start setting up the environment, let's clone the tutorial sources and set the `TUTORIAL_HOME` environment variable to point to the root directory of the tutorial:

[#cloneRepo]
[source,bash,subs="attributes+,+macros"]
----
git clone {github-repo} &&\
export TUTORIAL_HOME="$(pwd)/kubernetes-tutorial"
cd $TUTORIAL_HOME
----
copyToClipboard::cloneRepo[]

[#env-path]
=== Env and PATH

Download https://kubernetes.io/docs/setup/minikube[minikube] and add it to your path.

MacOS and Linux
----
export MINIKUBE_HOME=$TUTORIAL_HOME;
export PATH=$MINIKUBE_HOME/bin:$PATH
export KUBECONFIG=$MINIKUBE_HOME/.kube/config
export KUBE_EDITOR="code -w"
----

[#start-minikube]
=== Start Minikube

There are series of commands as steps that allows you get started with minikube. The following section shows how to start minikube with minimal and required configurtions:

[#minikube-set-cpu]
[source,bash,subs="+macros,+attributes"]
----
minikube --profile {tutorial-namespace} config set cpus 2 # <.>
----
copyToClipboard::minikube-set-cpu[]
<.> Using 2 cpus which is good enough for the basics
There is nothing in this series of exercises that is CPU intensive 

[#minikube-set-memory]
[source,bash,subs="+macros,+attributes"]
----
minikube --profile {tutorial-namespace} config set memory 6144 # <.>
----
copyToClipboard::minikube-set-memory[]
<.> Memory matters, especially when running traditional Java apps (e.g. Weblogic, Spring)

[#minikube-set-disk]
[source,bash,subs="+macros,+attributes"]
----
minikube --profile {tutorial-namespace} config set disk-size 50g # <.>
----
copyToClipboard::minikube-set-disk[]
<.> Its required that we give additional disk space for storing container images

[#minikube-set-vmdriver]
[source,bash,subs="+macros,+attributes"]
----
minikube --profile {tutorial-namespace} config set vm-driver {vm-driver} <.>
----
copyToClipboard::minikube-set-vmdriver[]
<.> Using virtualbox because it is available on all platforms.  There a number of 
https://kubernetes.io/docs/tasks/tools/install-minikube/#install-a-hypervisor[hypervisor options] that you can choose with minikube

[#minikube-set-k8sversion]
[source,bash,subs="+macros,+attributes"]
----
minikube --profile devnation config set kubernetes-version v1.17.0 #<.>
----
copyToClipboard::minikube-set-k8sversion[]
<.> Setting the Kubernetes version explicitly so you know what you are getting

[#minikube-set-profile]
[source,bash,subs="+macros,+attributes"]
----
minikube start --profile devnation #<.>
----
copyToClipboard::minikube-set-profile[]
<.> Profiles allow you to keep different configurations
